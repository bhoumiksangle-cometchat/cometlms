## Application Use Case

A full-stack Learning Management System where students learn from instructor-created courses with structured video lessons, quizzes, and certifications. Every part of the communication layer — course discussions, peer messaging, live sessions, AI tutoring, moderation, and support — is built natively into the platform, making this a self-contained, deeply integrated education product.

---

## Problem Statement

Most LMS platforms treat communication as an afterthought — basic comment sections, fragile video embeds, or no real-time features at all. The result is a disengaged learning experience where students can't get help fast, instructors can't connect with learners, and community never forms.

This LMS solves that by making communication a first-class feature:
- Every course has a live, moderated discussion channel
- Students and instructors can connect 1:1 via DMs and voice/video calls
- AI agents provide instant tutoring and FAQ assistance 24/7
- The backend stays in sync with all communication events via internal event bus
- Admins can moderate content at scale using built-in AI moderation

## Communication Feature Map

This section documents how each communication feature is built natively.

### 1. Real-Time Connection & Auth
**Where:** `apps/web/src/lib/socket.ts` + backend `chat/socket.server.ts`

- Initialise Socket.IO client on app load, authenticated with the user's JWT
- On user login/register, backend creates a chat identity record in Postgres
- Client connects to the Socket.IO namespace with `{ auth: { token } }`
- On logout, socket is disconnected and presence is updated

---

### 2. Course Discussion Groups
**Where:** Course Player page — Discussion tab

- When an instructor **publishes a course**, the backend automatically creates a **chat room** in the `ChatRooms` table with:
  - `roomId`: `course-{courseId}`
  - `name`: course title
  - `type`: `group`
  - `ownerId`: instructor ID
- When a student **enrolls in a course**, the backend adds them to the room's membership table
- On the course player, the Discussion tab renders the custom `<CourseDiscussion roomId="course-{id}" />` component
- Features: **typing indicators, read receipts, reactions, @mentions, message threading, file sharing, rich media**
- When a student is **refunded or unenrolled**, they are removed from the room membership

---

### 3. Direct Messages — Student ↔ Instructor
**Where:** Course page sidebar + Student/Instructor dashboard

- Full `<DirectMessages />` panel embedded in the dashboard
- Students can message the course instructor directly from the course player
- Instructors see all their student DMs in one inbox with unread counts
- Features: **read receipts, typing indicators, message reactions, file attachments, voice messages, AI smart reply suggestions**
- Instructors can mark conversations as resolved (custom status field on the conversation record)

---

### 4. Live Session / Office Hours (Voice & Video Calling)
**Where:** Instructor dashboard → "Start Office Hours" + Course page → "Join Live"

- Instructor clicks **"Start Office Hours"** → backend creates a video call session (via Daily.co or mediasoup WebRTC)
- A live session notification is pushed to all enrolled students via Socket.IO + push notification
- Students see a **"Join Live"** banner on the course page and join the call room
- Call features: **HD video, screen sharing, mute/unmute, call recording, participant list**
- After the session ends, the backend logs session duration and stores the recording URL
- 1:1 office hours: Instructor or student initiates from the DM screen with `<CallButtons />`

---

### 5. AI Study Assistant
**Where:** Floating widget on every course page + My Courses dashboard

- A **Study Assistant** is powered by the OpenAI API (GPT-4o) or a LangChain RAG agent indexed on course content
  - System prompt: "You are a helpful AI tutor for [Course Name]. Answer questions about the course content, help debug code examples, and explain concepts clearly."
- Students open the floating widget and chat with the assistant
- The assistant has access to: course syllabus, lesson descriptions, quiz question bank (via RAG tool)
- Features: **real-time streaming responses, tool calls, conversation history, typing indicators**
- Each course gets its own agent instance with course-specific context

---

### 6. Course FAQ Bot
**Where:** Course detail page (pre-enrolment) + Course discussion group (pinned bot)

- A **FAQ Bot** is added as a participant in every course discussion group
- Students can @mention the bot (`@FAQBot`) for instant answers
- When the bot is mentioned, the backend intercepts the event, calls OpenAI, and posts the reply back into the chat room
- FAQ data source: course description, syllabus, instructor-curated Q&A pairs stored in Postgres
- The bot is also embedded on the public course detail page as a pre-sales assistant

---

### 7. Instructor AI Copilot
**Where:** Instructor dashboard — Q&A Manager panel

- Instructors in the Q&A panel see **AI Smart Replies** suggested above the reply input (generated server-side via OpenAI)
- A **Copilot** can draft full replies to student questions — instructor reviews and sends
- **Conversation Summary** button on any long discussion thread → AI summarises the key questions and answers
- The Copilot also surfaces common unanswered questions from course discussions to the instructor dashboard

---

### 8. AI Content Moderation
**Where:** All course discussion groups, DMs between users

- Custom moderation pipeline on the message processing layer:
  - **Profanity filtering** — auto-delete or flag messages with slurs/hate speech (OpenAI Moderation API or custom wordlist)
  - **Image moderation** — reject NSFW images before delivery (AWS Rekognition or OpenAI Vision)
  - **Spam detection** — flag repeated/promotional messages with a rate-limit + similarity check
  - **Custom rules** — block sharing of external links (anti-piracy for course content)
- Flagged messages are logged to `ChatModerationLog` and surfaced in the admin queue
- Admin dashboard shows flagged message queue with approve / delete / ban actions
- Instructors can report messages from the UI; admins review and take action

---

### 9. Real-Time Presence & Typing Indicators
**Where:** Throughout the chat UI (handled by Socket.IO events)

- Online/offline presence shown on user avatars in DM list and course group member list
- Typing indicator emitted via `typing:start` / `typing:stop` Socket.IO events
- Presence changes update a `last_seen_at` field in Postgres for analytics
- Student activity dashboard shows "last active" derived from presence events

---

### 10. Message Reactions, Threads & @Mentions
**Where:** Course discussion groups, live session chat

- Students can react to instructor announcements (👍🔥❤️) in course group channels
- Questions in course discussions can be **threaded** — replies are nested under a parent message
- Instructors can @mention specific students in replies — parsed from message text server-side
- Admin can @mention the entire group (`@all`) for course-wide announcements
- Reaction counts shown on lesson discussion posts for engagement metrics

---

### 11. Push Notifications (Multi-Channel)
**Where:** Backend notification service (BullMQ + SendGrid + FCM)

- Push notifications via FCM / APNs for:
  - New DM from instructor
  - @mention in course discussion
  - Live session starting ("Instructor started office hours")
  - AI agent reply received
- Email notifications via SendGrid for:
  - Missed messages (offline users)
  - Weekly discussion digest
- Course-specific notifications (enrollment, certificates) handled via BullMQ + SendGrid

---

### 12. Internal Event Bus → LMS Backend Sync
**Where:** `apps/api/src/modules/chat/socket.server.ts`

All Socket.IO events are processed internally by the backend:

| Event | LMS Action |
|---|---|
| `message:sent` | Persist to DB; increment course engagement score |
| `message:edited` | Update message record |
| `message:deleted` | Soft-delete message record |
| `message:reaction_added` | Increment reaction count in analytics |
| `message:reaction_removed` | Decrement reaction count |
| `user:mentioned` | Trigger in-app notification to mentioned user |
| `user:presence_changed` | Update `last_seen_at` in Postgres |
| `user:blocked` | Log block action; alert admin if instructor blocks student |
| `group:member_banned` | Sync ban to LMS; revoke course access if needed |
| `group:member_joined` | Confirm enrollment sync |
| `call:ended` | Log call duration; store recording URL; mark session as completed |
| `call:meeting_ended` | Log group session details; notify attendees with recording link |
| `moderation:flagged` | Add to admin moderation queue; notify admin |
| `moderation:approved` | Clear flag from moderation queue |
| `message:read_by_all` | Mark instructor announcement as "seen by all" |

## User Workflows

### Student Workflow
1. Register → backend creates user record
2. Login → receive JWT; Socket.IO connects and authenticates
3. Browse course catalogue → view course detail → chat with FAQ Bot (pre-enrolment)
4. Enroll in course → backend adds student to course chat room
5. Open course player → Discussion tab shows live group chat
6. Ask question in discussion → @mention instructor or FAQ bot
7. Open AI Study Assistant widget → get instant help from AI agent
8. DM instructor directly from course page
9. Join live office hours → group video call
10. Complete course → receive certificate → share achievement in course discussion

### Instructor Workflow
1. Register → apply for instructor role → admin approves
2. Create course → on publish, backend creates course chat room + adds instructor as owner
3. Build curriculum → upload videos, create quizzes
4. Monitor Q&A tab → see student questions → use AI Copilot to draft replies
5. Use Smart Replies to quickly respond to common questions
6. View Conversation Summary of long threads to catch up
7. Start office hours → group video call → students join via course banner
8. After session, view recording and share link in course group
9. Moderate discussion — ban spam users from course group
10. View engagement analytics (message counts, reactions) from internal event data

### Admin Workflow
1. Login → Admin Dashboard (separate `/admin/*` route)
2. View platform KPIs including engagement metrics
3. Review moderation queue (messages flagged by AI Moderation)
4. Take action: approve / delete message / ban user from platform
5. Manage user accounts (suspend, force logout)
6. Configure AI agents (update prompts, swap LLM provider, enable/disable per course)
7. View activity event log and error rates

## Assumptions

- All real-time communication is handled in-house via Socket.IO; no third-party chat SDKs
- JWT is used for both REST API auth and Socket.IO handshake authentication
- Every LMS user gets a chat identity equal to their LMS user ID
- One chat room per published course; room ID = `course-{courseId}`
- AI agents are configured in the admin dashboard and connected via OpenAI or LangChain
- Voice/video calling is handled via Daily.co (pay-as-you-go) or self-hosted mediasoup
- AI moderation uses the OpenAI Moderation API and/or AWS Rekognition for image checks
- Push notification credentials (FCM/APNs) are configured in the backend environment
- Platform takes a configurable commission from course sales
- English only for Phase 1; translation feature is Phase 2

## Acceptance Criteria

### Real-Time Chat Init & Auth
- [ ] Socket.IO connects on app load without errors using JWT auth
- [ ] Auth token is never exposed in client-side storage; all auth is in-memory
- [ ] Chat session is in sync with LMS login/logout
- [ ] New users can send and receive messages within 2 seconds of registration

### Course Discussion Groups
- [ ] Chat room is created automatically when instructor publishes a course
- [ ] Student is added to room automatically on enrollment
- [ ] Student is removed from room on unenrollment/refund
- [ ] Messages in course discussion show typing indicators, read receipts, and reactions

### Direct Messages
- [ ] Student can initiate DM with instructor from course player
- [ ] Instructor sees unread message count in dashboard
- [ ] AI Smart Replies are shown as suggestions above the reply input

### Voice & Video Calls
- [ ] Instructor can start a group office hours call
- [ ] Enrolled students receive push notification and in-app banner
- [ ] Call end event is processed and logged within 5 seconds of call end
- [ ] Recording URL is stored in the LMS database post-call

### AI Study Assistant
- [ ] Agent responds within 3 seconds to student questions
- [ ] Agent correctly answers questions about the course syllabus and content
- [ ] Conversation history persists within the session

### Event Processing
- [ ] All 15 mapped internal events are processed and logged

- [ ] `moderation:flagged` events appear in admin queue within 10 seconds

### Moderation

- [ ] Admin can dismiss / delete / ban from moderation queue UI
